# App Analyzer - 競合アプリ分析・改善提案

WebアプリのURLを分析し、優れた機能やUI/UXパターンを抽出して、Memoriaの改善に活かします。

## When to Use This Skill

- 競合アプリや参考アプリを分析したい時
- 他のアプリの良い点をMemoriaに取り入れたい時
- UI/UXのベストプラクティスを学びたい時
- 新機能のアイデアを得たい時
- 「〇〇のようなアプリを作りたい」と言われた時

---

## 分析プロセス

### Step 1: アプリ情報の取得

```
入力: アプリのURL（例: https://example.com）

取得する情報:
1. ページ構造・ナビゲーション
2. 主要機能・ユーザーフロー
3. UI/UXデザインパターン
4. 技術スタック（可能な範囲で）
5. アクセシビリティ対応
```

### Step 2: 機能分析

以下の観点で分析：

| カテゴリ | 分析項目 |
|----------|----------|
| **ナビゲーション** | メニュー構造、ルーティング、ブレッドクラム |
| **認証・権限** | ログイン方式、権限管理、セッション管理 |
| **データ表示** | 一覧表示、詳細表示、検索・フィルタ |
| **データ入力** | フォーム設計、バリデーション、自動保存 |
| **フィードバック** | 通知、エラー表示、成功メッセージ |
| **パフォーマンス** | 読み込み速度、遅延読み込み、キャッシュ |
| **モバイル対応** | レスポンシブデザイン、タッチ操作 |

### Step 3: 改善提案の生成

```markdown
## 改善提案レポート

### 分析対象
- URL: [分析したURL]
- アプリ種別: [SaaS/教育/業務システム等]
- 分析日: [日付]

### 発見した優れた機能

#### 1. [機能名]
- **概要**: [機能の説明]
- **ユーザー価値**: [なぜ優れているか]
- **Memoriaへの適用案**: [具体的な実装提案]
- **実装難易度**: [低/中/高]
- **期待効果**: [改善される点]

#### 2. [機能名]
...

### UI/UXパターン

#### 1. [パターン名]
- **現在の実装**: [Memoriaの現状]
- **参考実装**: [分析アプリの実装]
- **改善案**: [具体的なコード/デザイン案]

### 推奨実装順序
1. [最優先: 理由]
2. [次点: 理由]
3. [将来的: 理由]
```

## 分析テンプレート

### 教育系アプリの分析ポイント

```
□ ポートフォリオ・成果物管理
  - アップロード方法（ドラッグ&ドロップ、カメラ撮影）
  - 整理・タグ付け機能
  - 振り返り機能
  - 共有・公開設定

□ 評価・フィードバック
  - ルーブリック
  - コメント機能
  - 相互評価
  - 進捗可視化

□ コミュニケーション
  - 教師-生徒間
  - 生徒同士
  - 保護者連携

□ データ分析
  - ダッシュボード
  - レポート出力
  - 傾向分析
```

### SaaS アプリの分析ポイント

```
□ オンボーディング
  - チュートリアル
  - 空の状態（Empty State）
  - プログレス表示

□ ダッシュボード設計
  - 情報の優先順位
  - カスタマイズ性
  - アクションへの導線

□ ワークフロー
  - タスク管理
  - 承認フロー
  - 通知設計

□ 設定・管理
  - ユーザー管理
  - 権限設定
  - カスタマイズ
```

## 技術スタック分析

### 確認方法

```bash
# ブラウザDevToolsで確認できる情報
- Network タブ: API構造、レスポンス形式
- Elements タブ: DOM構造、CSS Framework
- Application タブ: Cookie、Storage使用状況
- Lighthouse: パフォーマンス、アクセシビリティ
```

### 技術パターンの抽出

```typescript
// 分析結果のフォーマット
interface TechAnalysis {

    authentication: string;
    rateLimit: boolean;

    firstContentfulPaint: number;
```

## UI/UXパターン集

### 1. ナビゲーション

```
分析観点:
- グローバルナビの構成
- サイドバー vs トップナビ
- モバイルでのハンバーガーメニュー
- パンくずリストの有無
- 検索機能の配置
```

### 2. データ一覧表示

```
分析観点:
- テーブル vs カード vs リスト
- ページネーション vs 無限スクロール
- ソート・フィルタの操作性
- 一括操作（バルクアクション）
- 空の状態の表示
```

### 3. フォーム設計

```
分析観点:
- インライン vs モーダル vs ページ遷移
- リアルタイムバリデーション
- 自動保存
- エラー表示方法
- ステップフォーム
```

### 4. フィードバック

```
分析観点:
- トースト通知
- インライン通知
- ローディング表示
- スケルトンスクリーン
- オプティミスティックUI
```

## 分析例

### 例1: Notionの分析

```markdown
## 分析: Notion (https://notion.so)

### 優れた機能

1. **ブロックエディタ**
   - スラッシュコマンド（/）でブロック挿入
   - ドラッグ&ドロップで順序変更
   - インラインでのスタイル変更

   **Memoriaへの適用案:**
   - ポートフォリオ編集にブロックエディタを導入
   - 振り返りコメントの入力体験向上

2. **テンプレート機能**
   - ワンクリックでテンプレート適用
   - カスタムテンプレート作成

   **Memoriaへの適用案:**
   - 振り返りテンプレートの充実
   - 教師が独自テンプレートを作成可能に

### UI/UXパターン

1. **サイドバー**
   - 折りたたみ可能
   - お気に入り・最近のページ
   - ツリー構造のナビゲーション

2. **コマンドパレット**
   - Cmd+K で呼び出し
   - 全機能へのクイックアクセス
```

### 例2: Canvaの分析

```markdown
## 分析: Canva (https://canva.com)

### 優れた機能

1. **テンプレートギャラリー**
   - カテゴリ別整理
   - 検索・フィルタ
   - プレビュー機能

   **Memoriaへの適用案:**
   - ポートフォリオテンプレートギャラリー
   - カテゴリ: 自己PR、成果報告、振り返り等

2. **ドラッグ&ドロップエディタ**
   - 直感的な操作
   - スナップ機能
   - リアルタイムプレビュー

### パフォーマンス

- 画像の遅延読み込み
- プログレッシブ画像表示
- オフライン対応（PWA）
```

## AI Assistant Instructions

このスキルが起動されたら:

1. **URLを受け取る**
   - WebFetchツールでページ内容を取得
   - Playwrightでスクリーンショット・スナップショットを取得（必要に応じて）

2. **体系的に分析**
   - 上記テンプレートに沿って分析
   - 優れた点を具体的に抽出
   - Memoriaの現状と比較

3. **実装可能な提案を作成**
   - 具体的なコード例を含める
   - 実装難易度と優先度を明示
   - 段階的な導入計画を提案

4. **レポート形式で出力**
   - 分析結果サマリー
   - 具体的な改善提案
   - 参考コード/デザイン

Always:
- 具体的な実装案を提示（抽象的な提案に留めない）
- Memoriaの技術スタック（React/TypeScript/Tailwind）に合わせた提案
- 教育アプリとしての文脈を考慮

Never:
- 競合の悪口や批判
- 著作権を侵害するコードのコピー
- 非公開情報の推測
